# testing.md

## Цель тестирования

Проект должен иметь достаточное покрытие тестами для безопасного рефакторинга и расширения.

---

## Инструменты

- pytest
- httpx / FastAPI test client
- pytest-cov

---

## Типы тестов

### Unit tests
Проверяют:
- service
- utils
- отдельные функции преобразования данных

### Integration tests
Проверяют:
- API endpoints
- корректность статусов
- корректность response body
- фильтрацию
- пагинацию

---

## Требования

1. Каждый новый service должен иметь unit tests.
2. Каждый новый router должен иметь integration tests.
3. Для list endpoint должны быть тесты:
   - без фильтрации
   - с фильтрацией
   - с пагинацией
4. Для create / update / delete endpoint должны быть happy-path тесты.
5. При исправлении бага нужно добавить regression test.
6. Все тесты должны содержать короткие комментарии на русском языке там, где это помогает пониманию.

---

## Целевое покрытие

Минимальное покрытие: 80%
Желаемое покрытие: 90%
