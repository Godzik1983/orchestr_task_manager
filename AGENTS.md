# AGENTS.md

## Общие правила проекта

Проект написан на Python.
Стек: FastAPI + pytest.
Все комментарии в коде и поясняющие docstring должны быть на русском языке.
Если агент создает новый файл с кодом, он обязан добавлять комментарии и краткие пояснения в ключевых местах.

Проект должен следовать модульной структуре:
- src
- modules
- controllers
- routers
- middleware
- views
- interfaces
- utils
- tests/unit
- tests/integration

Все изменения должны быть согласованы с architecture.md и testing.md.

---

## Agent: Architect

Роль:
Отвечает за архитектуру проекта.

Обязанности:
- создавать и поддерживать согласованную структуру проекта
- обновлять architecture.md при изменении архитектуры
- следить, чтобы новые модули соответствовали общей схеме
- предлагать рефакторинг, если структура разрастается или дублируется код

---

## Agent: Backend

Роль:
Отвечает за реализацию backend-кода.

Обязанности:
- создавать роуты FastAPI
- создавать controllers
- создавать services
- создавать repositories
- создавать interfaces / schemas
- добавлять фильтрацию, пагинацию и валидацию
- писать комментарии на русском языке

---

## Agent: Testing

Роль:
Отвечает за тестирование.

Обязанности:
- создавать unit tests
- создавать integration tests
- добавлять или обновлять тесты после изменений в коде
- запускать тесты и исправлять несовместимости между кодом и тестами
- следить за покрытием не ниже целевого значения из testing.md

---

## Agent: Maintainer

Роль:
Отвечает за сопровождение проекта.

Обязанности:
- находить баги
- исправлять failing tests
- поддерживать README, architecture.md и testing.md в актуальном состоянии
- готовить осмысленные git commit message
- при необходимости готовить изменения для GitHub Actions
