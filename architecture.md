# architecture.md

## Цель проекта

Проект представляет собой backend-приложение для управления задачами.
Основная сущность: `Task`.

---

## Технологии

- Python
- FastAPI
- pytest
- SQLite (через стандартный модуль `sqlite3`)

---

## Архитектурный стиль

Используется слоистая архитектура.

Поток запроса:
`Router -> Controller -> Service -> Repository -> Database`

---

## Структура каталогов

```text
src/
  app.py                      - точка входа приложения
  modules/                    - доменные модули
  controllers/                - слой контроллеров
  routers/                    - слой роутов
  middleware/                 - middleware приложения
  views/                      - представления / response shaping
  interfaces/                 - Pydantic-схемы и интерфейсы данных
  utils/                      - вспомогательные утилиты (включая модуль БД)

tests/
  unit/                       - unit tests
  integration/                - integration tests
```

---

## Правила проектирования

1. Роуты не содержат бизнес-логику.
2. Бизнес-логика находится в `service`.
3. `Repository` отвечает только за доступ к данным.
4. Все новые endpoint должны сопровождаться тестами.
5. Все ключевые части кода должны иметь комментарии на русском языке.
6. Изменения архитектуры фиксируются в `architecture.md`.

---

## Модуль tasks

Модуль `tasks` поддерживает:
- создание задачи
- получение списка задач
- получение задачи по id
- обновление задачи
- удаление задачи
- фильтрацию
- пагинацию

Хранение данных:
- задачи хранятся в SQLite
- инициализация таблицы выполняется через `src/utils/db.py`
- слой `repository` работает через SQL-запросы
