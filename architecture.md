# architecture.md

## Цель проекта

Проект представляет собой backend-приложение для управления задачами с базовой аутентификацией пользователей.
Ключевые сущности: `Task`, `User`.

---

## Технологии

- Python
- FastAPI
- pytest
- SQLite (через стандартный модуль `sqlite3`)

---

## Архитектурный стиль

Используется слоистая архитектура.

Поток запроса:
`Router -> Controller -> Service -> Repository -> Database`

---

## Структура каталогов

```text
src/
  app.py                      - точка входа приложения
  modules/                    - доменные модули (tasks, auth)
  controllers/                - слой контроллеров
  routers/                    - слой роутов
  middleware/                 - middleware приложения
  views/                      - представления / response shaping
  interfaces/                 - Pydantic-схемы и интерфейсы данных
  utils/                      - вспомогательные утилиты (БД, безопасность)

tests/
  unit/                       - unit tests
  integration/                - integration tests
```

---

## Правила проектирования

1. Роуты не содержат бизнес-логику.
2. Бизнес-логика находится в `service`.
3. `Repository` отвечает только за доступ к данным.
4. Все новые endpoint сопровождаются тестами.
5. Все ключевые части кода имеют комментарии и docstring на русском языке.
6. Изменения архитектуры фиксируются в `architecture.md`.

---

## Модуль tasks

Модуль `tasks` поддерживает:
- создание задачи
- получение списка задач
- получение задачи по id
- обновление задачи
- удаление задачи
- фильтрацию
- пагинацию

Хранение данных:
- таблица `tasks` в SQLite
- инициализация выполняется через `src/utils/db.py`

---

## Модуль auth

Модуль `auth` поддерживает:
- регистрацию пользователя (`POST /auth/register`)
- вход пользователя (`POST /auth/login`)
- получение текущего пользователя (`GET /auth/me`)

Хранение данных:
- таблица `users` в SQLite

Безопасность:
- пароли хранятся в виде PBKDF2-хеша
- доступ реализован через JWT access token (HS256)
